### Turbopack FATAL panic 修复

**问题**：启动后访问任意页面报 `An unexpected Turbopack error occurred`，终端显示 `FATAL: Failed to write app endpoint /classroom/page`。

**根因**：项目根目录存在一个名为 `nul` 的普通文件（约 181 字节，内容是之前 CMD `dir > nul` 命令误创建的错误输出）。Windows 系统保留 `NUL` 为设备名，Turbopack 的 PostCSS 在处理 `globals.css` 时路径拼接到该文件，导致 `os error 1`（函数不正确）直接 panic。

**修复**：删除 `C:\develop\open\OpenMAIC\nul` 文件即可。

**顺带改动**（非根因，但已应用）：
- `next.config.ts` → `transpilePackages` 增加 `'three'`（修复 Three.js Turbopack 兼容性）
- `components/three-lab/three-scene.tsx` → OrbitControls import 路径改为 `three/addons/controls/OrbitControls.js`
- `app/page.tsx` → `<style jsx>` 改为 `page.module.css`（CSS Module），新增 `app/page.module.css`

**诊断方法**：查看 `%TEMP%\next-panic-*.log`，Caused by 段落会直接指向问题文件。

## MySQL + Drizzle ORM 用户数据库

**需求**：将用户信息从 localStorage 迁移到 MySQL，实现服务端认证。

**技术选型**：MySQL 8.x + Drizzle ORM + bcryptjs + HMAC token

**新建文件**：

| 文件 | 说明 |
|------|------|
| `lib/db/schema.ts` | Drizzle schema（users, otp_tickets 两张表）|
| `lib/db/index.ts` | MySQL 连接池（mysql2/promise + drizzle）|
| `lib/db/seed.ts` | 种子数据（预置测试账号 13800138000）|
| `lib/server/auth-token.ts` | HMAC token 生成/验证（格式：userId.timestamp.signature）|
| `lib/server/auth.ts` | 认证全流程：密码哈希、OTP 创建/验证、cookie 管理、用户查询 |
| `app/api/auth/send-code/route.ts` | POST 发送验证码 |
| `app/api/auth/register/route.ts` | POST 注册（OTP + 密码 + 自动登录）|
| `app/api/auth/login/route.ts` | POST 登录（密码/验证码二选一）|
| `app/api/auth/logout/route.ts` | POST 退出 |
| `app/api/auth/me/route.ts` | GET 当前用户（cookie 验证）|
| `drizzle.config.ts` | Drizzle Kit 配置 |
| `components/session-init.tsx` | 应用启动时从 /api/auth/me 恢复登录态 |
| `.env.local` | DATABASE_URL=mysql://openmaic:openmaic_pass@localhost:3306/openmaic |

**重构的文件**：
- `lib/store/session.ts` — 从 Zustand persist 本地存储改为异步 fetch API 调用，不再存 users[]/otpTickets
- `app/login/page.tsx` — 所有认证操作改为 async API 调用
- `app/register/page.tsx` — 同上
- `app/account/page.tsx` — 从 store 读取 userCreatedAt/userLastLoginAt（不再查 users 数组）
- `app/layout.tsx` — 添加 `<SessionInit />` 启动时恢复 session
- `docker-compose.yml` — 添加 MySQL 服务 + healthcheck
- `.env.example` — 添加 DATABASE_URL + AUTH_SECRET
- `package.json` — 添加 drizzle-orm, mysql2, bcryptjs 依赖 + db:* scripts

**认证机制**：
- 密码：bcrypt 哈希
- Token：HMAC-SHA256 签名，存 httpOnly cookie `openmaic_auth`，7天过期
- OTP：60s 冷却 + 5min 过期 + 5次尝试上限 + 预置账号 bypass

**启动步骤**：
1. `docker-compose up -d mysql`（启动数据库）
2. `npx pnpm db:push`（推送 schema）
3. `npx pnpm db:seed`（插入测试账号）
4. `npx pnpm dev`（启动应用）

**TypeScript 编译**：零错误

## 数据库启动与验证码显示修复

**问题**：MySQL 容器未启动导致 API 500 错误；注册时验证码错误（前端未展示 debugCode）

**修复**：
1. 启动 MySQL 容器 → `docker-compose up -d mysql`
2. 推送 schema → `DATABASE_URL=... npx drizzle-kit push`
3. 灌种子数据 → `DATABASE_URL=... npx tsx lib/db/seed.ts`
4. `.env.local` 补充 `AUTH_SECRET=<随机hex>`
5. `app/register/page.tsx` / `app/login/page.tsx` — `handleSendCode` 回调中把 `debugCode` 展示给用户（演示环境无真实短信）

**注意**：drizzle-kit 不会自动读 `.env.local`，需手动传 `DATABASE_URL` 环境变量

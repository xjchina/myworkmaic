## 错题本删除功能修复（关键 Bug 修复）

### 问题根因
`readMistakeNotebookEntries()` 从 `quizResults:*` localStorage key **实时重建**错题列表，完全不看 `mistakeNotebook:v1` store。而所有删除函数只操作 `mistakeNotebook:v1` store。结果：**删了→reload→从 quizResults 重新生成→删除无效**。

此外，`writeMistakeNotebookEntriesForScene()` 每次评分时会把**所有错题**重新写入 store，覆盖掉用户已删除的条目。

### 修复方案 — Deny-List 架构重构
将 `mistakeNotebook:v1` 的存储模型从**完整数据副本**改为**排除名单(deny-list)**：

- **Store 结构变更**：`{ entries: MistakeNotebookEntry[] }` → `{ removedKeys: string[] }`
  - `removedKeys` 存储用户主动删除的 `"sceneId::questionId"` 字符串数组
- **读取逻辑** (`readMistakeNotebookEntries`)：仍然从 `quizResults:*` 实时构建，但返回前用 `removedKeys` 过滤掉已删除的
- **写入逻辑** (`writeMistakeNotebookEntriesForScene`)：**不再往 store 写数据**（store 现在只存删除标记）
- **删除逻辑** (4个函数)：全部改为向 `removedKeys` 数组追加被删除的 key
- **清除逻辑** (`clearSubmitted` / `clearAllForScene`)：清理对应 sceneId 前缀的 removedKeys

### 修改文件
- `lib/quiz/persistence.ts`：
  - `MistakeNotebookStore` 接口: `entries` → `removedKeys`
  - `readNotebookStore()`: 适配新结构，过滤非法 key
  - `readMistakeNotebookEntries()`: 新增 deny-list 过滤步骤
  - `writeMistakeNotebookEntriesForScene()`: 简化为空操作（不再写 store）
  - `removeMistakeEntry()`: 改为 append key 到 removedKeys
  - `removeMistakeEntriesByScene()`: 改为批量 append 到 removedKeys
  - `removeMistakeEntries()`: 改为去重后 batch append 到 removedKeys
  - `clearAllMistakeEntries()`: 改为清空 removedKeys 数组
  - `clearSubmitted()` / `clearAllForScene()`: 同步清理 removedKeys 中对应 scene 的条目
